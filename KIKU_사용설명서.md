# KIKU 회원 관리 앱 사용 설명서

## 접속 방법

| 환경 | 방법 |
|------|------|
| PC | `KIKU_바로가기.html` 더블클릭 또는 브라우저에서 직접 URL 입력 |
| 핸드폰 | 브라우저에서 URL 직접 접속 또는 북마크 |

**앱 URL**
```
https://eunsolkang00.github.io/kiku-manager/kiku_manager_drive.html
```

---

## 시작하기 (매번 접속 시)

1. 페이지 접속
2. 상단 **Google 로그인** 버튼 클릭
3. 구글 계정 선택 (처음 이후엔 계정 선택 없이 바로 연결)
4. 자동으로 드라이브에서 데이터 불러오기 완료

> 로그인 후 상단 상태바에 "자동 불러오기 완료 — 회원 N명, 벙 N개" 뜨면 정상

---

## 상단 바 버튼 설명

| 버튼 | 기능 |
|------|------|
| Google 로그인 | 구글 계정 연결 및 데이터 자동 불러오기 |
| 로그아웃 | 구글 연결 해제 |
| 드라이브에 저장 | 현재 데이터를 구글 드라이브에 저장 |
| 드라이브에서 불러오기 | 드라이브에서 데이터 수동 불러오기 |
| 로컬 백업 | JSON 파일로 내 컴퓨터에 백업 저장 |

> **작업 후에는 반드시 "드라이브에 저장"을 눌러야 해.** 안 누르면 데이터가 사라져.

## 탭별 기능 설명

### 대시보드

앱을 열면 가장 먼저 보이는 화면이야.

- **지표 카드 4개**: 전체 회원 수 / 유령 대상 / 연락 완료 / 정상 활동 인원 한눈에 확인
- **다음 유령 정리일 알림**
  - 7일 이상 남음 → 파란색 안내
  - 7일 이내 → 노란색 경고
  - 당일 → 빨간색 긴급 알림
- **유령 판정 대상자 미리보기**: 현재 유령 대상자 목록 간략히 표시
- **최근 벙 현황**: 가장 최근 벙 3개 표시

---

### 회원 명단

전체 회원을 관리하는 탭이야.

#### 회원 추가
1. 우측 상단 **회원 추가** 버튼 클릭
2. 이름, 가입일 입력
3. **추가** 클릭

#### 회원 수정
- 해당 회원 행의 ✏️ 버튼 클릭
- 이름, 가입일, 최근 참여일 수정 가능
- 최근 참여일은 벙 참석 입력 시 자동 업데이트되지만, 필요 시 직접 수정 가능

#### 회원 삭제
- 해당 회원 행의 🗑️ 버튼 클릭
- 삭제 시 해당 회원의 벙 참석 기록도 함께 삭제됨

#### 운영진 연락 여부
- 회원이 운영진에게 연락했을 경우 체크박스 체크
- 체크 시 유령 판정에서 제외되고 "연락 완료" 상태로 변경
- 유령 정리일에 초기화 실행 시 전체 해제됨

#### 회원 상태 종류
| 상태 | 의미 |
|------|------|
| 🟢 정상 | 최근 2달 내 벙 참여함 |
| 🔵 신규 | 가입한 지 2달 미만 |
| 🟣 연락 완료 | 유령 기준 해당되나 운영진에게 연락함 |
| 🔴 유령 대상 | 세 조건 모두 충족 — 퇴출 대상 |

---

### 벙 관리

벙(모임) 기록을 관리하는 탭이야.

#### 벙 추가
1. **벙 추가** 버튼 클릭
2. 벙 이름, 날짜 입력
3. 참석자 체크박스 선택
4. **추가** 클릭
5. 선택한 회원의 최근 참여일이 자동으로 업데이트됨

#### 참석 수정
- 벙 카드의 ✏️ **참석 입력** 버튼 클릭
- 참석자 추가/제거 가능
- 저장 시 전체 회원의 최근 참여일 자동 재계산

#### 벙 삭제
- 벙 카드의 🗑️ 버튼 클릭
- 삭제 시 해당 벙의 참석 기록도 함께 삭제되고 최근 참여일 재계산됨

---

### 유령 정리

유령 회원을 판정하고 조치하는 탭이야.

#### 유령 회원 기준 (세 조건 모두 충족 시)
1. 기산일 기준 **2달 이전** 가입자
2. 기산일 기준 **이전 두 달간** 벙 참여 없음
3. 기산일 이전 **운영진에게 연락 없음**

#### 기산일
- 매 **짝수 달 1일** (2월 1일, 4월 1일, 6월 1일, 8월 1일, 10월 1일, 12월 1일)
- 앱이 자동으로 다음 기산일을 계산해서 표시함

#### 유령 정리일 절차 (짝수 달 1일마다)
1. **유령 정리 탭** 접속
2. 퇴출 대상 목록 확인 (각 회원별 조건 1/2/3 통과 여부 표시)
3. 카카오톡 등에서 퇴출 대상자 퇴장 처리
4. **회원 명단 탭**에서 해당 회원 삭제
5. 연락 완료 회원은 체크박스 확인
6. **초기화 실행** 버튼 클릭 (운영진 연락 여부 전체 리셋)
7. **드라이브에 저장** 클릭

> 초기화 버튼은 기산일 당일에만 활성화됨

## 데이터 저장 구조

- 데이터는 구글 드라이브에 `kiku_data.json` 파일로 저장됨
- 로그인한 구글 계정의 드라이브에 저장되므로 다른 기기에서도 동일한 데이터 사용 가능
- 로컬 백업 버튼으로 추가 백업 권장 (특히 유령 정리일 전후)

---

## 여러 운영진이 함께 쓸 때

- 구글 클라우드 콘솔 → OAuth 동의 화면 → 테스트 사용자에 운영진 이메일 추가
- 단, 데이터는 **로그인한 계정의 드라이브**에 저장되므로 한 명이 대표로 관리하는 방식 권장
- 여러 명이 동시에 수정하면 데이터가 덮어써질 수 있음

## 자주 있는 실수

| 상황 | 해결 |
|------|------|
| 작업 후 저장 안 누름 | 드라이브에 저장 버튼 클릭 |
| 불러오기를 안 해서 빈 화면 | 드라이브에서 불러오기 클릭 |
| 회원 삭제했는데 벙 기록에 남음 | 삭제 시 자동으로 제거됨, 정상 |
| 유령 대상인데 실제로는 다음 벙 참석 예정 | 해당 벙을 미리 등록하고 참석자로 체크해두면 유령 판정 제외됨 |
